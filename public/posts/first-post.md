# How I forced AI to write Vanilla CSS instead of Tailwind

Создание визуального редактора — сложная бизнес-задача. Но еще сложнее заставить LLM-модели строго следовать заданным правилам архитектуры.

## Проблема с "галлюцинациями" стилей
По умолчанию современные модели (особенно DeepSeek, Claude и Gemini) обожают использовать Tailwind CSS. Это логично: в их обучающей выборке миллионы проектов на React используют именно этот фреймворк. 

Даже если в системном промпте написать "Используй только Vanilla CSS", модель может по привычке добавить в JSX классы вроде `className="flex justify-center text-white"`. Для генератора кода это катастрофа: если в сгенерированном проекте не установлен Tailwind, эти классы просто не сработают, и дизайн рассыплется.

## Наше решение: Жесткие архитектурные промпты
В CRUSH AI мы решили эту проблему на уровне **Master Node** (Оркестратора). 

Вместо того чтобы просто просить ИИ "написать код", мы динамически собираем мега-промпт на основе того, какие ноды подключены к графу. Если пользователь выбрал в настройках ноды `React App` шаблон `vanilla`, Оркестратор встраивает в запрос жесткую директиву:

```javascript
// Пример того, как мы формируем правила под капотом
let stylingRule = "";

if (templateChoice === 'vanilla') {
    stylingRule = "CRITICAL STYLING RULE: You MUST use pure Vanilla CSS. DO NOT use ANY Tailwind CSS classes anywhere. Create standard .css files, write standard CSS, and import them in your components.";
}
```

### Мультимодальность (Vision) спешит на помощь
Но настоящий прорыв случился, когда мы подключили мультимодальные модели к процессу генерации интерфейсов. 

Теперь мы передаем ИИ скриншот желаемого дизайна (напрямую через ноду `Image / UI Design`) и заставляем его анализировать цвета, отступы и размеры напрямую с пикселей, а затем транслировать это в чистый CSS. Никаких слепых угадываний.

Будущее разработки — это не текстовые чат-боты, в которые нужно вставлять огромные куски кода. Это визуальные графы, где архитектура сама контролирует качество того, что пишет ИИ.
